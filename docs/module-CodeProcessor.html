<static> getResults

Gets the results of the code processor. Note: the results are sanitized to remove hidden properties and functions, thus allowing the results to be JSON.stringify()'d.

Parameters:
Name Type Argument Description
files Array[String] A list of files to process.
pluginList Array[String] A list of plugins to load. Each name must correspond with a subdirectory containing a node module in the 'plugins' directory.
logger Winston Logger A logger instance from the CLI (can be a CLI logger, which wraps winston, or a winston logger directly)
cli Object The configuration from the CLI
Properties
Name Type Description
sdks Object The list of all installed SDK
options Object <optional>
 Options for controlling the code processor
Properties
Name Type Argument Description
invokeMethods Boolean <optional>
 Indicates whether or not methods should be invoked. Default is true.
evaluateLoops Boolean <optional>
 Indicates whether or not to evaluate loops. Default is false.
maxLoopIterations Number <optional>
 The maximum number of iterations a loop can iterate before falling back to an unknown evaluation (as if evaluateLoops were false). This prevents code with extremely intensive code or code with infinite loops from taking down the system. Default is 10000.
maxLoopIterations Number <optional>
 The maximum number of iterations a loop can iterate before falling back to an unknown evaluation (as if evaluateLoops were false). This prevents code with extremely intensive code or code with infinite loops from taking down the system. Default is 10000.
maxRecursionLimit Number <optional>
 Indicates the maximum recursion depth to evaluate before erroring (infinite recursion guard)
executionTimeLimit Number | undefined <optional>
 Indicates the maximum time the app is allowed to run before erroring. Undefined means no limit.
exactMode Boolean <optional>
 Indicates whether or not the app should be evaluated in exact mode. Exact mode does not use ambiguous mode and throws an exception if an Unknown type is encountered.
nativeExceptionRecovery Boolean <optional>
 Indicates whether or not to try and recover from native exceptions. Default is false because enabling it has the potential of introducing errors in the analysis.
sdkPath String <optional>
 The path to the Titanium SDK to use
platform String <optional>
 The name of the platform being compiled for
modules Object <optional>
 A dictionary of module ids as keys. If the module is a common js module, the value is the path to the module entry point, otherwise the value is null.
