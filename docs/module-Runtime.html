Type:

Properties:
Name Type Description
invokeMethods boolean Indicates whether or not methods should be invoked
evaluateLoops boolean Indicates whether or not loops should be evaluated
maxLoopIterations number Indicates the maximum number of loop iterations to evaluate before erroring (infinite loop guard)
maxRecursionLimit bumber Indicates the maximum number of recurses to allow before claiming a recursive call is infinitely recursive (infinite recursion guard)
cycleDetectionStackSize number The size of the stack to use for cycle detection
maxCycles number The maximum number of cycles to allow
logConsoleCalls boolean If enabled, all console.* calls in a user's code are logged to the terminal
executionTimeLimit number | undefined Indicates the maximum time the app is allowed to run before erroring. Undefined means no limit
exactMode boolean Indicates whether or not the app should be evaluated in exact mode. Exact mode does not use ambiguous mode and throws an exception if an Unknown type is encountered
processUnvisitedCode boolean When set to true, all nodes and files that are not visited/skipped will be processed in ambiguous mode after all other code has been processed. While this will cause more of a project to be analyzed, this will decrease accuracy and can generate a lot of false positives.
